Reframe Backend Architecture

This document is a code-level backend map for investor/engineering due diligence. It describes service boundaries, request lifecycle, core runtime processes, and the full route inventory currently registered in the codebase.

1. System Overview

Execution Model

Single Node.js Express API server with modular route registration and shared middleware stack.

Express JWT/Cookie Auth SQLite/Postgres-compatible DB layer Rate Limits Audit Log

Frontend/Backend Contract

Frontend (React + Vite dashboard) consumes JSON APIs for auth, projects, AI, SEO, publishing, settings, billing, team/orgs, and admin operations.

Security Stack

Helmet defaults + CSP, SSRF-protected fetch path, request-id correlation, owner-only admin routes, auth middleware, and scoped rate limits by operation class.

AI Orchestration

Multi-provider model routing (Claude/Gemini/Groq/Ollama/auto-router), per-call credit checks, model capability detection, and guarded rewrite/export actions.

2. Runtime Processes

Auth Lifecycle

Register/login/refresh/logout + email verification + password reset + optional 2FA + profile/avatar/export/delete.

Project Lifecycle

Create/import/update/archive/restore/duplicate + pages CRUD + versions + workflow stage + activity + share links.

AI Rewrite Lifecycle

Validate prompt/scope -> estimate/check credits -> run provider rewrite -> return diff/state -> log usage.

Export/Delivery Lifecycle

Validate source -> build artifact (multiple targets) -> async export job optional -> cloud upload -> deployment history.

Publishing Lifecycle

Publish targets -> preview token generation -> deployment execution -> history retrieval -> rollback.

Org/Team Lifecycle

Organizations + members + invites + API key vault entries + owner/admin policy enforcement.

Billing Lifecycle

Stripe packages/invoices/checkout/portal/webhook + credit wallet balance/transactions/top-up/admin migration.

Admin Lifecycle

Owner-only health/audit/metrics + user management (ban/unban/reset/delete/plan/credits/create).

3. Module Boundaries

Core entrypoint

server/index.js bootstraps app, middleware, static serving, AI/export endpoints, and route registrations.

Auth

server/auth.js contains account lifecycle + Google OAuth + 2FA paths.

Projects

server/projects.js handles import preview, CRUD, pages, versions, shares, comments, approvals, search.

Publish

server/publish.js controls targets, previews, publish/rollback/custom-domain.

SEO

server/seo.js provides audit/generation/suggestions/snippet/project meta operations.

AI Studio/Assistant

server/assistant.js provides chat + studio runs APIs.

Settings/Org/Credits

server/settings.js, server/organisations.js, server/credits.js.

Infra adapters

cloudStorage, stripe, screenshot, googleServices, templates, translationMemory, presets.

5. Non-route Background/Support Processes

Job Queue

Asynchronous export tasks (`enqueueJob`/`getJobForUser`) with polling endpoint on export jobs.

Credit Accounting

Balance checks + deductions gate expensive AI/export actions before execution.

Audit + Metrics

Structured request log with request-id, status, duration and user context, plus admin export/usage metrics.

Static Delivery

Serves dashboard dist build, reset-password pages, share views, thumbnails and publish preview pages.
