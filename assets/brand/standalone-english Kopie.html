Reframe GTM v2.0

The Migration Engine for the Modern Web.

Stop rebuilding sites from scratch. Reframe ingests legacy URLs and reconstructs them into production-ready blocks instantly.

The Problem

Web agencies lose 70% of their project margin in the "rebuild" phase. Moving a client from an old PHP site to Shopify or WordPress requires 40+ hours of manual transcription, asset downloading, and CSS matching.

The Solution

A semantic ingestion engine that reverse-engineers the DOM. Reframe doesn't just "copy" a site; it understands the structure, localizes assets, and compiles a project into the target platform's native syntax.

Founder Story

That agility is our DNA: we build for the person in the trenches, not the manager in the boardroom.

Under the Hood: Technical Audit

Our architecture transitions from messy, unstructured HTML to a strictly typed Abstract Syntax Tree (AST).

1. Ingestion Layer

Headless browser scrapes DOM and neutralizing external scripts via Semantic Proxy.

2. AST Transformer

Unstructured nodes are mapped to Reframe Blocks. Styles are normalized to Tailwind/CSS vars.

3. Persistence

Project state stored in SQLite with full versioning snapshots for every AI edit.

4. Delivery Artifact

Compilers transform the AST into platform-specific code (Liquid, JSX, PHP).

Technical "How-to" Guides

Guide: How to Migrating a legacy WordPress site to Shopify in 3 steps

Step 1: Ingest. Paste the WordPress URL into Reframe. Our AST parser will separate headers, footers, and content sections automatically.

Step 2: Re-Brand. Use the "Brand Brain" tool to swap the legacy CSS colors with your new Shopify design system variables.

Step 3: Export. Select "Shopify Liquid" and download the zip. Upload it to your Shopify theme, and your sections are live—no manual coding required.
